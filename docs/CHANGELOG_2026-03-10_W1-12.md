# Changelog - 2026-03-10 (W1-12 Cash Reconciliation)

## Summary
Implemented W1-12 end-of-day cash reconciliation across backend API, Prisma schema, frontend operations UI, and integration coverage.

## Added
- New cash session API endpoints under `/api/v1/cash-sessions`:
  - `POST /api/v1/cash-sessions/open`
  - `POST /api/v1/cash-sessions/:id/close`
  - `GET /api/v1/cash-sessions`
- New Prisma model and enum for cash session lifecycle:
  - `CashSession`
  - `CashSessionStatus` (`OPEN`, `CLOSED`)
- New frontend route and page:
  - `/operations/cash-reconciliation`
  - Denomination-based close workflow with variance preview
- New backend integration test for open/duplicate/close/list lifecycle.

## Security and Access
- Open and close actions require `ADMIN` or `MANAGER` role.
- Tenant isolation is enforced in cash session queries.
- Open/close operations are audit logged.

## Calculation Details
- On close, system cash is derived from opening float plus completed CASH payments in the session window.
- Variance formula:
  - `varianceCents = declaredCashCents - systemCashCents`

- This push includes generated build artifacts currently tracked in `backend/dist` and `frontend/dist`, plus updated test output artifact in `backend/test-results`.
